# Блог ArendaVmeste — как добавлять статьи

## Структура

- Страница списка постов: `blog/index.html`
- Публикации: `_posts/YYYY-MM-DD-slug.md`
- Настройки: `_config.yml` (можно задать общие параметры сайта)

## Шаблон поста

```markdown
---
layout: post
title: "Заголовок"
description: "Короткое описание для превью"
reading_time: 5 мин чтения
permalink: /blog/slug/
excerpt_separator: <!--more-->
---

Первый абзац текста, который пойдёт в превью.

<!--more-->

Основной текст статьи в Markdown.
```

**Важно:**

- `excerpt_separator` позволяет явно указать, какая часть текста попадёт в анонс на главной и на странице блога.
- `permalink` формирует человекочитаемый URL. Если не указать, Jekyll сгенерирует его автоматически из заголовка.
- `reading_time` отображается в списке постов, можно указать любой текст (например, «4 мин чтения»).

## Запуск локально

```bash
bundle exec jekyll serve
```

После этого открывайте `http://localhost:4000` и проверяйте, что пост появляется на главной и в `/blog/`.

## Как писать

- Используйте Markdown: заголовки (`##`), списки, цитаты.
- Для выделения цифр и ключевых мыслей используйте жирное (`**текст**`).
- Избегайте HTML внутри статьи, если это не обязательно.
- Если нужна таблица или сложная разметка — Markdown сходится с HTML, можно вставлять блоки `<div>` / `<table>`.

## Изображения и медиа

- Сохраняйте изображения в `assets/images/` (создайте папку при необходимости).
- Подключайте их в посте через `![alt]( {{ '/assets/images/file.jpg' | relative_url }} )`.

## Проверка перед публикацией

- Просмотрите анонс на главной: нет ли слишком длинного первого абзаца.
- Убедитесь, что URL без опечаток.
- Посмотрите на мобильной версии (в DevTools) — карточки блога адаптивные.

Готово! Новый пост сразу попадёт на сайт при следующем деплое или сборке.
